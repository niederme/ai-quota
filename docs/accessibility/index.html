Accessibility

Last updated: April 7, 2026

AIQuota aims to be straightforward to use on macOS and on the web. Accessibility is an ongoing part of how the product and site are designed and updated.

How we approach accessibility

We try to support clear type, visible focus states, readable color contrast, keyboard access, and motion that respects system preferences.

Website considerations

The AIQuota site is built as simple semantic HTML with lightweight CSS and JavaScript. Interactive elements are intended to be keyboard reachable, and the hero video respects reduced-motion preferences.

App considerations

AIQuota is designed to keep important status visible at a glance while still exposing more detailed information in the app. We aim to make menu bar, popover, and widget experiences understandable and stable.

Ongoing work

Accessibility is a continuous effort, and some areas may not yet work perfectly for every device, setting, or assistive technology combination.

Feedback

If you run into an accessibility issue or need help using AIQuota, email help@aiquota.app.
